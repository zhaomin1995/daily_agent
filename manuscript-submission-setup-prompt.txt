## File 6: web/src/app/api/submissions/route.ts

```typescript
import fs from "fs";
import path from "path";
import { SUBMISSIONS_DIR } from "@/lib/submissionPaths";
import yaml from "js-yaml";

export const dynamic = "force-dynamic";

export async function GET() {
  if (!fs.existsSync(SUBMISSIONS_DIR)) return Response.json([]);
  const files = fs.readdirSync(SUBMISSIONS_DIR).filter((f) => f.endsWith(".yaml"));
  const submissions = files.map((f) => {
    const content = fs.readFileSync(path.join(SUBMISSIONS_DIR, f), "utf-8");
    return yaml.load(content);
  });
  return Response.json(submissions);
}

export async function POST(request: Request) {
  const body = await request.json();
  const id = body.id || `manuscript-${Date.now()}`;
  const filePath = path.join(SUBMISSIONS_DIR, `${id}.yaml`);
  fs.mkdirSync(SUBMISSIONS_DIR, { recursive: true });
  fs.writeFileSync(filePath, yaml.dump({ id, ...body }));
  return Response.json({ ok: true, id });
}
```

## File 8: web/src/app/api/submissions/[id]/generate/route.ts

POST endpoint that generates documents. Body: `{ type: "cover-letter" | "author-block" | "contributor-statement" | "checklist" }`

**cover-letter**: Generate markdown cover letter using:
- Manuscript title, journal name
- Corresponding author name + email from coauthors.yaml
- A template: "Dear Editors, We are pleased to submit our manuscript '[title]' for consideration in [journal]. [Novelty placeholder — user fills in]. All authors have read and approved the manuscript. The authors declare no conflicts of interest. Sincerely, [Corresponding Author Name]"

**author-block**: Format author list as:
```
First Author¹, Second Author¹², Third Author²
¹ Institution One, Department
² Institution Two, Department
Corresponding author: Name, email
```
Number affiliations automatically, deduplicate shared affiliations.

**contributor-statement**: Generate CRediT statement in standard format:
"[Name]: Conceptualization, Formal analysis. [Name]: Writing – review & editing." etc.

**checklist**: Return the appropriate checklist items as a JSON array with `{ item: string, checked: boolean, note: string }` objects. Implement STROBE (22 items), CONSORT (25 items), and PRISMA (27 items) as static arrays.

For STROBE checklist items use the official STROBE 2007 checklist for observational studies (cohort, case-control, cross-sectional).

## Design guidelines

- Match the existing app's design language: zinc color palette, rounded-xl cards, text-sm, Tailwind CSS 4
- Status badges: use the same pill style as action item source badges
- All forms should auto-save on blur (PATCH to the API), no explicit Save button needed
- Use optimistic UI updates — update local state immediately, then sync to server
- The /submissions page should feel like a lightweight project tracker, not a complex form wizard
- Mobile-friendly: all grids should collapse to single column on small screens
- Keep all generated documents as plain markdown — no PDF generation needed

---

## What NOT to build

- Do not build any browser automation or form-filling for journal portals
- Do not build email sending — all outputs are copy/paste ready
- Do not build authentication — this is a local personal tool
- Do not add a database — YAML files are the source of truth
